- [内容] **案例不用VIP群成员**。改用"很多朋友"的通用表述，避免编造具体会员案例。
- [内容] **股价数据必须准确**。生成前需天哥确认标的和价格区间；可以请天哥发 OptionStrat 截图作为数据依据。
- [内容] **Covered Call 段落需区分"防御性"与"正常操作"**。天哥平时强调高位卖 Call，低位防御性卖 Call 是特殊情景，必须在脚本中明确说明，避免与历史内容矛盾。
- [风格] **所有期权比喻统一使用房地产类比**：正股 = 全款买房，LEAPS = 临时产权/订金，卖 Call = 收房租，Spread = 押金差额做二房东。以后生成脚本时自动应用此统一比喻体系。
- [流程] **截图工作流已确认**：天哥将 OptionStrat 截图保存到 `screenshots/` 文件夹，Claude 读取后提取真实数据生成脚本。生成前先确认标的 + 请天哥发截图。

### Roll 展期脚本修改反馈（2026-04-07）

- [风格] **不要称呼观众为"复利玩家"**。开场直接说"大家好"即可，不需要花哨称呼。
- [风格] **不要自称"天哥"**。脚本中用"我"即可，避免第三人称自称。
- [风格] **比喻要直观不牵强**。"时光机"太牵强，"调节器"更直接。比喻选择标准：观众一听就懂，不需要额外解释。
- [内容] **数据推演优先用真实实盘截图**。不用假设数据，直接用 Discord 实盘操作截图做分步拆解（多少钱买回、多少钱卖出、差价多少）。
- [内容] **Roll 次数建议两次，但理论上可以无限 Roll**。不要说"最多两次"的硬性上限。实在不行只 Roll Out 也行。
- [结构] **不要在脚本中预告后续章节名称**。比如不要说"后面的风控铁律部分"，直接说"视频后面还有详细的解法"。
