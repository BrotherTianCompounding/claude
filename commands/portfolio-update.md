## 第一步：解析持仓 CSV

天哥会提供两个 Fidelity 持仓 CSV 文件：
- **本周持仓**（最新，作为当前状态）
- **上周持仓**（上周，用于对比变化）

### 账户总市值计算方法
Fidelity CSV 中每行有 `Percent Of Account` 字段。用大仓位反推：
```
总账户市值 = 该持仓当前市值 ÷ 该持仓占比百分比
```
用至少两个大仓位交叉验证（结果应接近，取平均）。

### 必须从 CSV 提取的数据清单

| 数据项 | 提取方式 |
|--------|---------|
| 本周总市值 | 用百分比反推（见上） |
| 上周总市值 | 同上，从上周 CSV 反推 |
| 账户变化（金额+%） | 本周 - 上周 |
| QQQ 周涨跌幅 | 对比两份 CSV 中 QQQ 的 `Last Price` |
| 本周新开 Sell Put | 对比两份 CSV，找负数量（Quantity < 0）且上周不存在的期权 |
| 新 Sell Put 权利金 | 从 `Cost Basis Total` 字段反推（负期权的 cost basis = 原始收到的权利金） |
| 浮亏最大的 Sell Put | `Total Gain/Loss Dollar` 最负的负数量期权 |
| 浮亏最大的 LEAPS | `Total Gain/Loss Dollar` 最负的正数量 Call 期权（到期日 > 6个月） |
| 底仓总浮盈 | VOO + SPY + NVDA + FXAIX 的 `Total Gain/Loss Dollar` 加总 |
| SPAXX 现金余额 | SPAXX 行的 `Current Value` |

### 按天哥的三层仓位系统解读持仓

天哥的仓位配置逻辑如下，解读 CSV 时按此分类：

```
第一层：定投底仓（目标 58%，封顶 70%）
  ├── 30% → QQQM
  ├── 30% → VOO
  └── 40% → 个股（单只 ≤ 10%）
  标的：VOO、QQQM、SPY、FXAIX、NVDA、GOOGL、TSLA（正股部分）

第二层：现金仓位（有收入 5%，无收入 10%）
  └── 存放在 SPAXX 货币基金

第三层：期权仓位（剩余）
  ├── 80% → 轮子策略（Sell Put + Covered Call）
  │         现金做 Cash-Secured Put
  │         被接盘后做 Covered Call
  └── 20% → LEAPS / 合成多头（可再卖 Call 收租）

现金流循环：Sell Put 权利金 → 买 LEAPS → LEAPS 卖 Call →
           补充现金仓位 → 按比例定投到底仓 → 复利滚大
```

**Margin 铁律**：只在熊市 + 不打算行权 + ≤25% Margin 下才可用。其他情况一律不用。

### 本期主角股选择（每期聚焦 3 只，不要每只都讲）

| 主角角色 | 选择规则 | 优先标的 |
|---------|---------|---------|
| Sell Put 主角① | 本周新开的 Sell Put，权利金最高的 | TQQQ、HOOD、SOXL |
| Sell Put 主角② | 当前浮亏最大或最需要 Roll 的 Sell Put | HOOD、APP、NVDA Put |
| LEAPS 主角 | 浮亏或浮盈最显著的 LEAPS Call | TSLA 2028、QQQ 2027 |

其他持仓在账户总览部分一笔带过，不需要每个都详细讲。

## 写作铁律（来自 feedback_log，每次必须应用）

1. **开场**：只用「那么废话不多说，Let's Go！」，不加长版自我介绍段落
2. **比喻**：所有期权比喻统一使用房地产类比（正股=全款买房、LEAPS=临时产权/订金、Sell Put=卖保险、Covered Call=收租）
3. **数字来源**：所有数字必须来自 CSV 真实数据，不允许估算或编造
4. **浮亏展示**：浮亏仓位必须诚实展示，不美化，这是天哥公开账户的核心差异化
5. **CC 区分**：如涉及 Covered Call，低位防御性使用时必须注明「防御性操作」，区别于平时高位卖 Call
6. **案例表述**：不用 VIP 群成员作为案例，改用「很多朋友」的通用表述
7. **Margin 铁律**：脚本中涉及 Margin 时，必须注明只在熊市、不超过 25%、不打算行权的前提下使用
