# 视频导演标注系统

## Overview

接收已完成的文案脚本，输出两样东西：
1. **完整标注版脚本** — 每一句话都标明用 A-ROLL 还是 B-ROLL，B-ROLL 注明具体类型和内容
2. **设计文档** — 针对需要制作的 PPT 页面，给出详细设计说明供美工执行

---

## 可用素材类型

| 类型 | 代号 | 适用场景 |
|------|------|----------|
| 主播出镜 | `A-ROLL` | 情绪表达、观点输出、直接对观众说话 |
| 历史交易截图 | `B-ROLL 历史交易截图` | 展示实际持仓、账单、盈亏记录 |
| OptionStrat截图 | `B-ROLL OptionStrat` | 展示期权策略图、盈亏曲线、Greeks数据 |
| Moomoo K线图 | `B-ROLL Moomoo K线图` | 展示股价走势、技术面分析、K线形态 |
| PPT页面 | `B-ROLL PPT` | 概念解释、数据对比、关键结论、要点归纳、流程图、比喻可视化 |

**所有 B-ROLL 右下角：** 保留主播小头像（Picture-in-Picture）

## A-ROLL vs B-ROLL 决策规则

**用 A-ROLL 当：**
- 表达个人观点、情绪、态度（"我也是人，我的账户也有浮亏"）
- 直接号召行动（点赞订阅、加Discord）
- 过渡衔接句（"那么问题来了……"）
- 风险警告（需要眼神接触增加可信度）

**用 B-ROLL 当：**
- 提到具体数字、截图、账单 → 历史交易截图
- 解释期权策略参数、展示行权价/到期日/权利金/盈亏曲线 → OptionStrat截图
- 提到股价走势、K线形态、技术面分析 → Moomoo K线图
- 解释概念、展示数据对比、总结要点、流程图、比喻可视化 → PPT页面

**总体原则：** 只要嘴里说的东西"可以被画面具象化"，就切B-ROLL。A-ROLL留给纯粹的"人与观众对话"时刻。

## 设计文档格式

对每一个需要制作的 PPT 页面，在脚本标注完成后，附上对应的设计文档：

```markdown
---
### 设计文档：[素材名称]
**类型：** PPT页面
**对应脚本位置：** 第X部分，台词「……」处
**时长：** 约X秒

**画面内容：**
[具体描述画面里有什么元素、文字、数字、布局]

**配色/风格：**
[主色调、字体风格、是否与频道VI一致]

**文案内容（画面上显示的文字）：**
[直接给出画面上要出现的所有文字]
---
```

对于 OptionStrat截图 和 Moomoo K线图，无需设计文档，只需在标注标签中写清楚展示什么内容即可（天哥自行截图）。历史交易截图同理。

## 录制版排版规范

标注版脚本完成后，额外输出一份「录制版脚本」，规则如下：

1. **逐句断行**：每句话独占一行，以句号、问号、感叹号、省略号为断句依据。方便主播逐句录制、逐句对照。
2. **结束标签后断两行**：每个 `[A-ROLL X 结束]` 或 `[B-ROLL ... 结束]` 标签后面，必须空出**两个空行**（即视觉上有一整段空白），明确区分不同镜头段落。
3. **保留所有标签**：开始/结束标签原样保留，不做任何删减。

**示例：**
```
[A-ROLL 1 开始]
大家好，我是天哥。
说实话，刚刚过去的这个3月，整个美股市场简直就是开启了"地狱模式"。
我相信很多朋友这段时间连股票账户都不想打开。
[A-ROLL 1 结束]


[B-ROLL 历史交易截图 展示3月期权账单 开始]
大家看屏幕上这张图。
这是我3月份作为期权卖方操作的最终账单。
[B-ROLL 历史交易截图 展示3月期权账单 结束]


[A-ROLL 2 开始]
那么问题来了——我这近3万美金，到底是怎么赚到的？
[A-ROLL 2 结束]
```

## 标注行为规范

**必须做：**
- 每个标注块都有明确的开始和结束标签
- B-ROLL 标签里写清楚「展示什么」，不要只写类型
- 需要设计的素材，每一个都出设计文档，不能遗漏

**不要做：**
- 不要把整段话都标成A-ROLL，能切B-ROLL就切
- 不要设计过于复杂的PPT页面，以实用清晰为主
- 设计文档里的文案直接可用，不要留"待填写"占位符

## 文件输出规范

完成录制版脚本后，自动保存到 `D:\ai_claude_projects\youtube\output\` 目录下：

1. 以视频标题创建文件夹（去掉标点，空格可用或省略），例如：`3月全市场割肉我靠SellPut静静收了近3万美金/`
2. 录制版脚本保存为 `录制版脚本.md`
3. 设计文档如有需要可单独保存为 `设计文档.md`

如果文件夹或文件不存在则创建新的，如果已存在则更新。
