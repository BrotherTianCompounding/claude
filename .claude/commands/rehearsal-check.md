# 彩排审核 (Rehearsal Check)

## Overview

模拟一位财经博主拿着录制版脚本 + B-ROLL 素材清单，对着每一段 B-ROLL 画面"试讲"一遍。发现脚本台词与画面素材之间的不匹配、不可执行、信息顺序错误等问题。

**触发时机：** 脚本 + B-ROLL 导演标注完成之后、正式录制之前，必须跑一遍。

---

## 审核输入

需要两份文件：
1. **录制版脚本**（含 B-ROLL 标注标签）
2. **设计文档 / PPT 文件**（知道每页 PPT 长什么样）

如果没有设计文档，根据标注标签中的描述推断画面内容。

---

## 审核流程

按脚本顺序，逐段 B-ROLL 执行以下四项检查：

### 检查一：素材可执行性（能不能做出来？）

对每一段 B-ROLL，问自己：

**如果是 PPT 页面：**
- PPT 上显示的文字/数字/图表，脚本里有没有完整给出？还是需要自己编？
- PPT 内容是否自洽？数字有没有算错？

**如果是 OptionStrat / 券商期权链截图：**
- 脚本描述的是**当前真实可截图的数据**，还是**假设的未来场景**？
- 如果是假设场景（比如"假设股价跌到 XX 后卖 Covered Call"），这个截图在现实中截不到——标记为 ❌ 不可执行
- 如果需要特定日期/行权价/权利金的截图，这些参数是否明确到可以直接去截图？

**如果是 Moomoo K线图：**
- 需要标注什么指标？（均线、RSI、成交量等）
- 需要标注特定的信号点吗？（比如"RSI 跌破 50 的那天"）这个信号点是过去的真实数据还是假设？

**如果是历史交易截图：**
- 截图存在吗？日期、金额、标的是否与脚本描述完全一致？

### 检查二：台词-画面匹配度（观众看到的和听到的对不对得上？）

- 当主播在讲某个数字/概念时，画面上是否正在展示对应的内容？
- 有没有"嘴里在讲 A，画面还停在上一段的 B"的情况？
- 同一段 B-ROLL 的时长是否合理？（太多台词挤在一页 PPT 上 = 观众看不过来）
- 建议每页 PPT 对应的台词不超过 30-40 秒。超过的建议拆分。

### 检查三：信息顺序（观众到这里知道什么？）

这是最关键的一项检查。站在**第一次看视频的观众**角度，按时间顺序走：

- 当前这段台词引用的概念/术语/数字，观众在前面的内容里已经见过了吗？
- 如果引用了后面才会介绍的内容（前向引用），标记为 ⚠️ 顺序问题
- 典型错误：
  - 用了仓位比例做计算，但仓位比例在后面的章节才介绍
  - 提到某个策略名词，但那个策略还没讲过
  - 引用了"上期视频"的内容但没有简短复述，新观众跟不上

### 检查四：假设场景 vs 真实数据

脚本中经常会用假设场景来教学（"假设 TQQQ 跌到 40…"）。问题是：

- 假设场景对应的 B-ROLL 是什么？如果标注的是 OptionStrat 截图，你截不到假设场景的数据
- **解法建议**：假设场景应该用 PPT 展示（可以自由设计画面），而不是用需要真实数据的截图类素材

## 输出格式

```
## 彩排审核报告

### 逐段检查结果

**[B-ROLL 标签名称]**
- 可执行性：✅ / ❌ + 原因
- 台词匹配：✅ / ⚠️ + 具体问题
- 信息顺序：✅ / ⚠️ + 观众此时还不知道什么
- 假设vs真实：✅ / ⚠️ + 是否需要换素材类型

（每段 B-ROLL 都要检查，没问题的可以合并简写）

---

### 问题汇总（按严重程度排序）

**❌ 不可执行（必须改）：**
1. [问题描述] → [建议改法]
2. ...

**⚠️ 顺序/匹配问题（强烈建议改）：**
1. [问题描述] → [建议改法]
2. ...

**💡 优化建议（可选）：**
1. [问题描述] → [建议改法]
2. ...

---

### 修改行动清单

按脚本顺序列出所有需要修改的地方：
1. [脚本位置] — [改什么] — [怎么改]
2. ...
```

## 审核行为规范

**必须做：**
- 逐段检查，不跳过任何一个 B-ROLL 标注块
- 站在"拿着脚本去录制"的角度思考：我真的能对着这个画面说出这段话吗？
- 区分"不可执行"（截不到图、做不出来）和"不合理"（能做但观众看了会困惑）
- 给出具体可执行的修改建议，不只说"这里有问题"

**不要做：**
- 不要审核脚本内容本身的对错（那是 video-script-review 的工作）
- 不要重新编排 B-ROLL（那是 video-director 的工作）
- 不要审核 PPT 的视觉设计（那是 ppt-designer 的工作）
- 如果某段没问题，直接写 ✅ 即可，不要硬凑问题
