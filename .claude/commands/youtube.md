## 天哥的仓位配置系统（生成脚本前必须理解）

> 这是天哥真实账户的仓位管理逻辑，每次生成实盘相关脚本时，必须按此系统解读持仓截图，不能乱说仓位比例。

### 三层仓位结构

```
总资产 = 定投仓位 + 现金仓位 + 期权仓位
```

#### 第一层：定投仓位（Index/Stock DCA）
- **比例计算**：年龄 + 20%（天哥今年38岁 → 目标 58%）
- **封顶**：70%（50岁及以上封顶在70%，不再递增）
- **内部配比**：
  - 30% → QQQM
  - 30% → VOO
  - 40% → 个股（单只个股最多不超过总账户10%）

#### 第二层：现金仓位
- **有工资/收入时**：保留 5%
- **无工资/收入时**：保留 10%
- **用途**：轮子策略的现金担保（Cash-Secured Put 的底层资金）

#### 第三层：期权仓位
- **比例**：总资产 - 定投仓位 - 现金仓位 = 剩余（约 32–37%）
- **内部配比**：
  - 80% → 轮子策略（Wheel Strategy）
    - 用现金或个股做 Sell Put / Covered Call
  - 20% → 长期看涨（LEAPS 或合成多头/PMCC）
    - 可以在 LEAPS 或合成多头上卖 Call，产生额外现金流

### 现金流循环逻辑（核心）

```
现金/个股
    ↓ Sell Put → 收权利金
权利金积累
    ↓ 购买 LEAPS / 合成多头
LEAPS / 合成多头仓位变大
    ↓ 卖 Call 收租（PMCC）
期权总仓位扩大
    ↓ 多余资金填充现金仓位
现金仓位充足
    ↓ 按 QQQM:VOO:个股 = 30:30:40 比例定投
定投仓位持续增长 → 复利滚大
```

> 核心理念：期权产生的现金流，最终流回定投仓位，形成"现金流 + 复利"双引擎增长。

### Margin 使用原则（铁律）
- 尽量 **不用 Margin**，Sell Put 优先使用现金担保（Cash-Secured Put）
- 唯一例外：熊市 + 明确不打算行权 + 只使用不超过 **25% Margin** 来 Sell Put
- 普通行情绝对不开 Margin
