# 百万之路 — 每周视频大纲生成器

## Overview

按「交易日志」十章模板逐章提问，收集本期视频所需的全部信息，输出一份可以直接拿来录制的视频大纲。

**不写完整脚本。** 天哥会对着 outline 自己录音发挥。

---

## 前置准备

每次启动前，按顺序读取：
1. `style_guide/tiange_style_guide.md`
2. `templates/weekly_1M_Journal_outline.md` — **本 skill 的核心模板**
3. `feedback/feedback_log.md`

---

## 执行流程

按模板十章顺序逐章提问。**每次只问一章**，等天哥回答后再问下一章。

每轮开始时先问：「这部分你有信息可以提供吗？还是我来帮你查？」

---

### 第1轮：基础信息

```
这期是第几期？日期？实盘第几周？
```

---

### 第2轮：一、开场钩子 HOOK

```
这周最戏剧性或最有意思的事情是什么？
HOOK 风格你想用哪种？
  A) 数字冲击 — "这周我赚了 $X,XXX 权利金"
  B) 浮亏坦白 — "这周账户掉了 $XX,XXX"
  C) 市场波动 — "VIX 飙到了 XX，我做了这件事"
  D) 行权惊喜 — "XXX 被 assign 了，但我其实挺高兴"
  E) 自定义（你来说）
```

---

### 第3轮：二、市场背景 MARKET CONTEXT

```
这周市场数据你有吗？（SPY/QQQ/VIX/10Y 涨跌、重大事件）
有的话直接给我；没有的话我来查。
```

**如果天哥没有数据：** 自动执行完整市场调研（等同 `/market-brief` 级别），搜索并填入：
- SPY / QQQ 本周涨跌 %
- VIX 水平及状态
- 10年期美债收益率
- 本周显著事件
- 对天哥交易的影响分析

---

### 第4轮：三、账户仪表盘 PORTFOLIO DASHBOARD

```
上周净值多少？本周净值多少？
资产分配明细：防御侧 / 现金 / Wheel / LEAPS 各多少？
（有仪表盘截图可以直接发给我）
```

**自动计算：** 涨跌、总盈亏、百万进度、占比偏差。

---

### 第5轮：四+五、交易明细 + 专项情况

```
请上传本周的券商交易截图，我来帮你提取参数。
一共有几笔交易要讲？
```

**截图处理：** 收到截图 → 调用 `/trade-analyzer` 提取参数 → 确认 → 逐笔追问理由和风险计划。

**自动识别专项：** 展期 / 指派行权 / LEAPS 买入 → 填写对应专项模板。

**汇总权利金：** Puts + Calls + 展期净收益 = 总计 + 本周 ROI。

---

### 第6轮：六、本周教训 LESSON OF THE WEEK

```
这周最大的实操心得是什么？要跟实际交易挂钩。
（可以跳过）
```

---

### 第7轮：七、计分板 SCOREBOARD

```
累计数据：周数、总P&L、累计权利金、平均每周权利金、胜率、预计达标日期？
（有仪表盘截图也可以直接发）
```

---

### 第8轮：八、现金流瀑布 WATERFALL UPDATE

```
这个月有资金流动吗？权利金转入 / 定投？
没有就跳过。
```

---

### 第9轮：九、下周预演 + 催化剂

```
下周你有什么交易计划或观察名单吗？
```

**自动调研：** 下周经济日历 + 重要财报日程，询问天哥应对计划。

---

### 第10轮：十、CTA + 结尾

```
本期的互动话题想问观众什么？
```

**自动填充：** 实盘周数 + 账户金额 + 品牌 Slogan。

严格按 `templates/weekly_1M_Journal_outline.md` 十章结构输出完整大纲，末尾附录制后检查清单。

## 第二阶段：草稿 → 脚本整理

大纲完成后，天哥发文字草稿过来，Claude 整理成完整脚本：

1. **【先做】事实审核** — 用 WebSearch 验证草稿里所有具体数字、事件、专家言论、历史比较；发现错误先告诉天哥确认，不要静默修正
2. 按大纲章节重新组织，理清逻辑，补充遗漏
3. 保持天哥口语风格，润色但不改成书面语
4. **固定元素必须有（不能漏）：**
   - HOOK（开头完整话术）
   - 免责声明
   - VIP Discord 推广（固定话术）
5. 添加 `[展示]` B-Roll 标注
6. 自审数字和期权逻辑
7. 保存到 `output/百万之路EP[XX]/脚本.md`

## 规则

- **逐章提问，不要一次性问完** — 每次只问一章，等回答后再问下一章
- **每轮先问天哥有没有数据** — 有就用他的；没有就自行调研
- **天哥说"跳过"或"没有"就跳过** — 不强迫每个模块都有内容
- **截图优先** — 交易信息优先通过截图 + `/trade-analyzer` 提取
- **市场调研要完整** — 无数据时执行 `/market-brief` 级别的全面搜索
- **数字必须来自天哥提供的信息或可靠调研，不得编造**
- **自动计算能算的都算** — 涨跌百分比、百万进度、年化回报、占比偏差等
- **固定元素绝对不能漏** — HOOK、免责声明、VIP Discord 推广，每期必须有
